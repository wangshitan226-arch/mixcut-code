1. **轮询500错误**：点击"开始生成"后，前端轮询 `GET /api/renders/dh_render_xxx` 返回500，因为该端点不存在

### 【Bug修复】

#### 【修复1】后端启动入口错误 - app.py 未注册 Blueprint

- **问题**：运行 `python app.py` 启动的是旧版单体架构（约1355行），未注册 `digital_human_bp`、`kaipai_bp`、`ai_bp` 等 Blueprint，导致所有数字人、声音克隆、快排相关API返回404
- **根因**：项目存在两套架构——`app.py`（旧版单体）和 `app_new.py`（模块化），但实际运行的是旧版
- **修复**：将 `app.py` 改为转发入口，委托给 `app_new.py` 的模块化架构
- **文件**：`backend/app.py`

修改前（1355行单体代码）：
```python
app = Flask(__name__)
db = SQLAlchemy(app)
# ... 所有路由直接定义在app.py中 ...
if __name__ == '__main__':
    app.run(host='0.0.0.0', port=3002, debug=True)
```

修改后（14行转发入口）：
```python
from app_new import create_app, start_cleanup_scheduler
app = create_app()
if __name__ == '__main__':
    start_cleanup_scheduler()
    from websocket import socketio
    socketio.run(app, host='0.0.0.0', port=3002, debug=False, allow_unsafe_werkzeug=True)
```

#### 【修复3】轮询500错误 - 添加 GET /api/renders/\<render\_id\> 端点

- **问题**：前端轮询 `GET /api/renders/dh_render_xxx` 返回500，因为后端只有 `GET /api/renders`（列表），没有 `GET /api/renders/<id>`（单条查询）
- **修复**：新增 `GET /api/renders/<render_id>` 端点，支持两种查询：
  1. 如果 render\_id 在 `render_tasks` 内存字典中（正在进行的任务），返回任务状态
  2. 如果 render\_id 在数据库 Render 表中（已完成的历史记录），返回完整渲染信息
- **特殊处理**：material\_ids 中可能包含 DigitalHuman ID（而非 Material ID），需查询 DigitalHuman 表获取素材信息
- **文件**：`backend/routes/renders.py`

```python
@renders_bp.route('/renders/<render_id>', methods=['GET'])
def get_render_by_id(render_id):
    if render_id in render_tasks:
        # 返回进行中的任务状态
        ...
    render = Render.query.get(render_id)
    if not render:
        return jsonify({'error': '渲染记录不存在'}), 404
    # 返回完整渲染信息，包含videoType
    ...
```

### 【已知问题】

#### OSS视频 ERR\_CONNECTION\_CLOSED

- **现象**：浏览器直接访问 `https://mixcut.oss-cn-hangzhou.aliyuncs.com/...` 报 `ERR_CONNECTION_CLOSED`
- **排查**：
  - PowerShell 直接测试 OSS URL 返回 200，说明文件存在且 Bucket 有公共读权限
  - OSS 控制台 CORS 已配置（来源 `http://localhost:3000`，允许 GET/POST/PUT/DELETE/HEAD，允许 Headers `*`，暴露 Headers ETag/x-oss-request-id）
  - 问题根源：浏览器安全策略（混合内容限制、Referer 检查等）导致直接访问外部 OSS URL 连接被重置
- **修复**：前端播放/下载外部 OSS URL 时，自动通过后端代理端点 `/api/proxy/video?url=xxx` 访问
  - `OptimizedVideoPlayer`：检测外部URL，自动拼接代理路径
  - `App.tsx` 完成弹窗：视频播放和下载链接同样使用代理
  - `ResultsScreen` 缩略图：外部OSS URL也走代理
- **文件**：`frontend/src/components/OptimizedVideoPlayer.tsx`、`frontend/src/App.tsx`、`frontend/src/components/ResultsScreen.tsx`

#### 【修复8】数字人视频缩略图不显示

- **问题**：作品列表中数字人视频没有缩略图，显示为空白或加载失败
- **根因**：
  1. DigitalHuman 创建时 `cover_url` 为 None（用户没有上传封面）
  2. 渲染管线创建 Render 记录时 `thumbnail=dh.cover_url`，因此也为 None
  3. 已有的缩略图URL是外部OSS URL，浏览器直接访问报 `ERR_CONNECTION_CLOSED`
- **修复**：
  - 后端新增 `_extract_thumbnail_from_video()` 函数：使用 FFmpeg 从视频URL提取第1秒帧，上传到用户自己的OSS bucket
  - 渲染管线3处创建Render记录时，当 `dh.cover_url` 为空，自动调用该函数提取缩略图
  - 前端缩略图为空时显示蓝紫渐变占位图 + User 图标
  - 前端缩略图外部URL走后端代理
  - 为已有的1条数字人Render记录补充了缩略图
- **文件**：`backend/routes/renders.py`、`frontend/src/components/ResultsScreen.tsx`
